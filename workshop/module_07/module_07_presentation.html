THE BUILT-IN TOOL

Diagnostics ship in the SDK.

Import diagnose, activate, and deactivate from fundamental.diagnostics. activate() and deactivate() turn diagnostics on and off globally; diagnose() scopes them to a single block.

What you get: a timestamped log file written into the directory you pass, automatic secret redaction so excerpts are safe to share, and on an exception an enhanced error report — traceback, SDK version, platform, and the trace_id — before the error re-raises.

diagnose() — scoped diagnostics.

Wrap one call. Auto-deactivate on exit. Safe to paste into a ticket.
module_07_diagnostics_debugging.ipynb · cell 11 
from fundamental.diagnostics import diagnose, activate, deactivate

# global on/off — writes a timestamped log file into log_dir
activate(log_dir="./logs")
proba = nexus.predict_proba(X_holdout)
deactivate()

# scoped — activates for the block, deactivates on exit
with diagnose(log_dir="./logs"):
    proba = nexus.predict_proba(X_holdout)
WHY THIS MATTERS

Surgical, not global.

activate() stays on until you call deactivate(). diagnose() scopes diagnostics to one block, deactivates on exit, and writes a timestamped log file into the directory you pass — ready to attach to a ticket.

WHAT GETS LOGGED

Every request. Every response. The trace id.

Endpoint, row count, model id, latency, HTTP status. Every NEXUS call writes a structured line through the fundamental logger hierarchy. Errors include the full exception class and message, plus the trace id the server assigned to the request.

API keys and other secrets are masked automatically. Log excerpts are safe to paste into a support ticket — and the trace id is the one line that routes the ticket straight to your request in the server logs.

THE ONE IDENTIFIER

Include the trace id. Everything else is optional.

A support ticket without a trace id means the engineer on the other end searches logs by timestamp and key prefix — and usually guesses wrong. A ticket with a trace id jumps straight to your exact request in the server logs.

The trace id is on every response and every error log line. diagnose() puts it in the file. DiagnosticHarness puts it in the DataFrame. The whole toolkit is downstream of that one rule: always capture the trace id, always include it.

THE CONTRAST

Without: slow.

Five fingerprints — the common three.

Symptom. Exception. First check. Most tickets you file will be one of these.
01

Bad API key

Symptom: 401 on the first call.
Exception: AuthenticationError.
First check: strip whitespace, verify length. Most auth failures are a trailing newline from a paste.

02

Rate limit

Symptom: rapid 429s.
Exception: RateLimitError.
First check: log timestamps under 2s apart means no backoff is wired up. Fix the retry code, not the key.

03

Poll timeout

Symptom: timeout during poll.
Exception: RequestTimeoutError.
First check: the job is still running. Retry the poll with the same task_id. Do not resubmit.

11 / 16

Five fingerprints — the other two.

Client-side validation and registry drift. Often mistaken for bugs; usually easy fixes.
04

Shape mismatch

Symptom: TypeError or ValueError on predict, before the network call.
Exception: TypeError / ValueError — raised client-side, before any API call.
First check: does X.shape[1] match the training feature count? Re-check the enriched frame — a dropped column or silent dedup often hides here.

05

Stale model id

Symptom: error on load.
Exception: NotFoundError.
First check: the model id is stale, deleted, or from the wrong environment. Verify against models.list() before you blame the code.

DiagnosticHarness — pipeline capture.

One dataclass. Every operation. A DataFrame you can attach to a ticket.
module_07_diagnostics_debugging.ipynb · cell 18 
@dataclass
class DiagnosticHarness:
    events: list = field(default_factory=list)
    def record(self, operation, status, duration_ms=None, error=None):
        self.events.append({
            "operation":      operation,
            "status":         status,
            "trace_id":     getattr(error, "trace_id", None),
            "error_type": type(error).__name__ if error else None,
        })
    def summary(self) -> pd.DataFrame: return pd.DataFrame(self.events)
# wrap any NEXUS call — success or failure
harness = DiagnosticHarness()
try:
    probas = nexus.predict_proba(X_holdout)
    harness.record("predict_proba", "success")
except Exception as exc:
    harness.record("predict_proba", "error", error=exc)
WHY THIS MATTERS

Every event, in one table.

harness.summary() is a pandas DataFrame — one row per operation, with status, trace_id, and exception type. Filter, sort, attach to an incident report. The output survives a process restart; the diagnostics do not.
